Ventura下安装HD4600/HD5000补丁教程

对于新版本的Ventura的发布，很多显卡已被删除，因此，在更新/安装Vemtura 13后出现显卡为驱动的问题（3MB），这里为大家做出驱动4代显卡的教学（HD4000/HD4600/HD5000等）

一.准备工作

1.OpenCore Legacy Pather（简称OCLP），这里用到的是V0.5.0（GUI）版本的；
2.OpenCore Configurator （或OCAuxiliaryTools）,这里下载网站请参考 opencore 初步配置 One 中的（4.6.2）OC编辑器 篇；

3.驱动AMFIExemption.kext(不强求)，具体方法请参考本文章的 二实际操作-2

二.实际操作

1.用 OpenCore Configurator 打开EFI-OC-Config.plist 文件 ；
2.在NVRAM -› 7C436110-AB2A-4BBB-A880-FE41995C9F82 -› boot-args中添加amfi_get_out_of_my_way=1启动参数（Clover与OC位置不同，在Boot -› Arguments处更改 项）；
3.在NVRAM -› 7C436110-AB2A-4BBB-A880-FE41995C9F82 -› csr-active-config改成FF0F0000（关闭SIP）；
4.重启系统后打开 OpenCore Legacy Pather，选择post install root patch，继续选择start root patching，等待加载，添加信任新扩展并重启；
5.点击左上角苹果图标 -› 关于本机 -› 显卡 处查看是否有显存出现，其他内容请参考后续-Hackintool解读

三.系统更改

Mac OS10.11以上版本，Apple为了系统环境安全，在核心中引入了SIP（System Integrity Protection），又称Rootless Mode机制，更改系统内容就要将该机制关闭才能操作，因此我们必须要关闭SIP

注意：更改SIP需要进入恢复模式**Recovery**操作，白苹果与黑苹果进入恢复模式的方式不同，请勿掺杂！

⑴白苹果

1.开机按住command+R键，进入恢复模式；

2.开机按住Option键，在出现的启动选择菜单中选择**RecoveryHD**进入恢复模式

⑵黑苹果

1.开机选择 Recovery版本号 直接进入恢复模式

Ⅰ .关闭SIP

1.在恢复模式下，点击实用工具 -› 终端；

2.在终端中输入命令：**csrutil disable** 并回车执行命令；

3.出现**Successfully disabled System Integrity Protection. Please restart the machine for the changes to take effect.**（成功关闭系统完整性保护。请重启机器以使更改生效。）时，则关闭SIP成功（系统完整性保护）；

4.重启，进入系统，在终端中输入**csrutil status** 回车执行命令；

5.System Integrity Protection status: disabled.则表示系统完整性保护功能已经关闭。

Ⅱ.开启SIP

1.在恢复模式下，点击实用工具 -› 终端；

2.在终端中输入命令：**csrutil enable** 并回车执行命令；

3.出现**Successfully enable System Integrity Protection. Please restart the machine for the changes to take effect.**（成功开启系统完整性保护。请重启机器以使更改生效。）时，则开启SIP成功（系统完整性保护）；

4.重启，进入系统，在终端中输入**csrutil status** 回车执行命令；

5.System Integrity Protection status: enabled.则表示系统完整性保护功能已经关闭。

注意:系统完整性保护主要作用是保护系统稳定性，安全性，建议更改后，及时重新开启该功能，不要长期处于关闭状态,以防造成不必要的损失。
